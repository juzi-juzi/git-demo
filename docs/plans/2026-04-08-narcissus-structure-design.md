# 三位水仙花程序结构整理 Design

**目标**

在不改变脚本输入输出行为的前提下，把 `5.py` 整理成更完整的结构。

**范围**

- 仍然只处理合法整数输入
- 仍然只判断三位水仙花数
- 保持终端运行时输出不变
- 增加 `main()` 作为脚本入口
- 允许模块被安全导入，不在导入时立刻读取输入

**方案对比**

1. 保持当前脚本式结构，仅调整注释
   改动最小，但脚本仍然在导入时执行输入输出，不利于复用和测试。

2. 提取 `main()` 并加入口保护
   保持外部行为不变，同时让判断逻辑和脚本入口分离，适合这个小程序。
   这是推荐方案。

**最终设计**

`5.py` 保留两个主要部分：

- `is_narcissus_number(number: int) -> bool`
  负责按三位水仙花数定义做纯判断
- `main()`
  负责读取输入并输出中文结果

文件底部使用 `if __name__ == "__main__": main()` 作为入口保护。

**测试**

至少覆盖：

- 现有黑盒脚本行为保持不变
- 模块可被导入，不在导入时触发 `input()`
- 模块暴露 `main()` 和 `is_narcissus_number()`
